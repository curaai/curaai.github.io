YOLO 란??

Object Detection 알고리즘 중 하나로서 faster r cnn과 달리 하나의 네트워크 구조를 가지고 있으며, processing 과정 때문에 작은 물체를 자주 detection하지 못한다는 단점이 있다. 하지만 아주 빠릅니다.

Model Architecture

Faster R CNN과 다르게 YOLO의 Model은 하나의 Network로만 이루어져 있다. 그래서 faster r cnn 보다 학습하는 과정도 단순화 되어있고, process speed도 더 빠르지 않은가 싶다.

여기서 가장 중요한 부분은 저기 빨간색 box 부분이다. 저 부분을 visual화 시켜보면 이런식으로 되어있다.

Grid Cell

기본적인 conv network를 거친후에 N x N x (B*5 + cls)로 변환을 한다.

  • 여기서 N은 위의 사진처럼 이미지를 N x N으로 나누는
    것이다.
  • (B*5 + cls)는 각각의 Grid Cell은 Bbox를 예측하는데 이 때 BBox의 개수를 B 라고 하고 + classification할 cls의 개수를 말한다.
  • 그리고 각각의 Bbox에 대해 Confidence score를 가지고 있다. confidence score는 Pr(Object) IOU로서 `(물체가 있을 확률) (Bbox 와 Ground Truth Box가 겹치는 정도 == IOU)이다. `
  • 그리고 cls에서 각각의 class에 해당하는 Object인지를 확인하며 이를 ‘conditional class probability’라 부른다. 글자가 기므로 CCP로 부르겠습니다.

Loss Function

  1. Object가 있는 Grid Cell i번째에 대해 Bbox j번째의 x, y를 계산
  2. Object가 있는 Grid Cell i번째에 대해 Bbox j번째의 width, height를 계산
  3. Object가 있는 Grid Cell i번째에 대해 confidence score를 계산
  4. Object가 없는 Grid Cell i번째에 대해 confidence score를 계산
  5. Object가 있는 Grid Cell i번째에 대해 CSCS를 계산
  • λcoord 는 좌표값에 대한 loss와 다른 loss의 균형을 맞추기 위한 balance parameter
  • λnoobj 는 object가 없는 경우가 많으므로 없는 경우와 있는 경우의 균형을 맞추기 위한 balance prameter
